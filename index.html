AVA Integration — ai-agent.local.yaml

Add this tool to your AVA config so it calls the knowledge base during conversations:

# Add inside your context in ai-agent.local.yaml contexts: default: prompt: >- You are Ava, an AI voice assistant. When a caller asks a question about our products, services, or policies, call the knowledge_base tool FIRST, then speak the answer you receive. tools: - name: knowledge_base type: http description: "Search company knowledge base for answers" url: "http://localhost:8001/search" method: GET parameters: q: type: string description: "The caller's question" required: true response_field: answer

Direct API endpoint (for testing or custom integration):

# Search the knowledge base GET http://localhost:8001/search?q=your+question+here # Returns: { "answer": "Spoken answer for AVA", "results": [...], "source_count": 2 }
